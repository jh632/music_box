# music_box 架构设计

## 项目概述

基于 ESP32-S3 的桌面音乐盒，播放本地音频文件，搭配 WS2812B 灯效和 OLED 信息显示。

```
硬件清单:
  ESP32-S3-N8R8 (8MB Flash / 8MB PSRAM)
  MAX98357A I2S 数字功放 → 4Ω 3W 喇叭
  WS2812B RGB 灯珠 (单总线)
  0.96" OLED (I2C, SSD1306)
  Boot 按键 (GPIO0, 保留)
  独立功能按键 x4 (GPIO11~14, 低有效)
```

---

## 三层架构

music_box 是单机设备，无通信协议、无多传感器协调，因此简化为 3 层 (GERB_GSI 的 5 层子集)：

```
┌────────────────────────────────────────────┐
│     应用层 (Application Layer)              │
│     播放控制、UI管理、LED灯效、按键处理       │
├────────────────────────────────────────────┤
│     设备驱动层 (Device Layer)               │
│     dev_audio / dev_led_strip /            │
│     dev_display / dev_button / dev_init     │
├────────────────────────────────────────────┤
│     硬件抽象层 (HAL Layer)                  │
│     GPIO / I2C / I2S / Timer / RMT / ADC   │
└────────────────────────────────────────────┘
          ↓
  ESP-IDF (底层驱动)
```

### 层间依赖规则

```
Application  →  Device  →  HAL  →  ESP-IDF
  ↓ 可调用同层或下层，禁止反向调用
```

- Application 不可直接调用 HAL (定时器管理除外，与 GERB 规则一致)
- Device 持有 HAL ops 指针，通过 `_get_ops()` 获取
- 每层对外暴露 `_init_all()` 聚合初始化

## 模块职责

### HAL 层

| 模块 | 职责 | 封装内容 |
|------|------|----------|
| `hal_gpio` | GPIO 初始化与电平控制 | `gpio_config`, `gpio_set_level` |
| `hal_i2c` | I2C 总线管理 | `i2c_master_init`, 读写寄存器 |
| `hal_i2s` | I2S 音频输出 | `i2s_driver_install`, `i2s_write` |
| `hal_timer` | 硬件定时器封装 | `esp_timer` 创建/启停/删除 |
| `hal_rmt` | RMT 外设 (WS2812B) | RMT 通道初始化 + 发送 RGB 数据 |
| `hal_adc` | ADC 采样 (光敏电阻) | `adc_oneshot_unit_init`, `adc_oneshot_read` |

### Device 层

| 模块 | 职责 | 依赖 HAL |
|------|------|----------|
| `dev_audio` | 音频播放控制 (播放/暂停/停止/音量) | `hal_i2s` |
| `dev_led_strip` | WS2812B 灯效 (常亮/呼吸/频谱同步) | `hal_rmt`, `hal_timer` |
| `dev_light_sensor` | 光敏电阻读取 + 滑动平均滤波 | `hal_adc` |
| `dev_display` | OLED 显示 (歌曲信息/状态栏) | `hal_i2c` |
| `dev_button` | 按键检测：自动/手动、切歌、音量控制、播放/暂停 | `hal_gpio` |
| `dev_init` | 聚合初始化，配置参数集中于此 | 所有 hal |

### Application 层

| 模块 | 职责 |
|------|------|
| `app_player` | 播放状态机 (IDLE → PLAYING → PAUSED)、曲目管理 |
| `app_ui` | OLED 界面更新 (歌曲名、进度、亮度等級) |
| `app_brightness` | 读取 dev_light_sensor → 计算目标亮度 → 驱动 dev_led_strip |
| `app_init` | 应用层聚合初始化 |

## main.c 编排

```c
// main.c — 只做层级编排，不出现任何具体模块的配置参数
void app_main(void) {
    // ESP-IDF 系统组件
    ESP_ERROR_CHECK(nvs_flash_init());

    // 分层初始化（顺序即依赖顺序）
    ESP_ERROR_CHECK(dev_init_all());   // 内部拉起 HAL 各模块
    ESP_ERROR_CHECK(app_init_all());

    ESP_LOGI(TAG, "Music Box started.");
}
```

## dev_init_all 聚合

```c
// device/src/dev_init.c
esp_err_t dev_init_all(void) {
    // I2C 总线最先初始化（OLED 依赖）
    ESP_RETURN_ON_ERROR(hal_i2c_init(I2C_NUM_0, GPIO_NUM_42, GPIO_NUM_41, 400000),
                        TAG, "i2c");

    ESP_RETURN_ON_ERROR(dev_audio_init(&audio_cfg),           TAG, "audio");
    ESP_RETURN_ON_ERROR(dev_led_strip_init(&led_cfg),         TAG, "led");
    ESP_RETURN_ON_ERROR(dev_light_sensor_init(&light_cfg),    TAG, "light");
    ESP_RETURN_ON_ERROR(dev_display_init(&disp_cfg),          TAG, "disp");
    ESP_RETURN_ON_ERROR(dev_button_init(&btn_cfg),            TAG, "btn");
    return ESP_OK;
}
```

```
GPIO11~14 按键中断 → dev_button → app_player / app_ui 状态切换
                         ↓
app_player 状态机 ───→ dev_audio 播放/暂停
    │                    ↓
    │               hal_i2s → MAX98357A → 喇叭
    │
    ├──→ app_brightness ──→ dev_led_strip (灯光效果同步)
    │        │                   ↓
    │   dev_light_sensor     hal_rmt → WS2812B
    │        ↓
    │   hal_adc (ADC1_CH0)
    │   ↑ 光敏电阻 AO
    │
    └──→ app_ui (OLED 更新)
            ↓
        dev_display → hal_i2c → OLED
```

## 状态机

```
           ┌──────────┐
           │  IDLE    │ 开机 / 播放列表空
           └────┬─────┘
                │ 播放/暂停按键 / 自动播放
                ▼
           ┌──────────┐
    ┌─────│ PLAYING  │─────┐
    │     └────┬─────┘     │
    │          │            │
    │ 切歌按键 │      曲目结束
    │ GPIO12   │            │
    │          ▼            ▼
    │     ┌──────────┐  ┌──────────┐
    │     │ PLAYING  │  │  IDLE    │
    │     │(下一首)  │  │(播放完毕)│
    │     └──────────┘  └──────────┘
    │
    │ 播放/暂停按键
    ▼
┌──────────┐
│ PAUSED   │  暂停播放
└──────────┘

GPIO11: 自动/手动模式切换（光敏控制）；GPIO12: 切歌；GPIO13: 音量控制；GPIO14: 播放/暂停；GPIO0: Boot 保留，不参与业务按键。
```

状态机实现于 `app_player.c`

## Device 层 API 接口设计规范

### 1. 设计模式：ops + handle

Device 层继承 HAL 层的 ops 模式，并引入 **不透明 handle** 支持多实例：

```
HAL:   _get_ops() → ops 单例 → 全局静态状态（每个外设仅一个实例）
Device: _get_ops() → ops 单例 → handle 管理实例状态（可能多个实例）
```

- **handle** = 不透明指针 (`typedef struct dev_xxx_s *dev_xxx_handle_t`)，由 `init()` 创建
- **ops** = 函数表结构体，通过 `dev_xxx_get_ops()` 获取单例
- 所有操作通过 `ops->method(handle, ...)` 调用
- `.c` 内所有函数 `static`，仅 ops 表对外暴露

### 2. 调用约定

```c
// 获取 ops 单例
const dev_audio_ops_t *audio = dev_audio_get_ops();

// init 创建 handle
dev_audio_handle_t h;
audio->init(&cfg, &h);

// 所有操作通过 handle
audio->play(h, "/music/track1.mp3");
audio->set_volume(h, 80);

// deinit 销毁 handle
audio->deinit(h);
```

### 3. 配置结构体规范

| 规则 | 说明 |
|------|------|
| 命名 | `dev_<module>_config_t`，仅子类型才加中间词 |
| 内容 | 封装所有硬件参数（GPIO、端口号、时序参数等） |
| 入口 | 通过 `init()` 传入，不允许在 `.c` 中硬编码 |
| 透明度 | `.h` 中不出现芯片型号（SSD1306、MAX98357A、WS2812B） |

### 4. 生命周期

```
dev_xxx_get_ops()         ← 获取 ops 单例（始终成功）
ops->init(cfg, &handle)   ← 创建实例，分配资源，初始化 HAL
ops->method(handle, ...)  ← 功能调用
ops->deinit(handle)       ← 释放实例，归还 HAL 资源
```

- `init` 失败时不得分配残留资源
- `deinit` 必须幂等（重复调用无害）
- handle 在 deinit 后由调用方置空

### 5. HAL 依赖方式

Device `.c` 内部持有 HAL ops 指针，通过 `_get_ops()` 获取：

```c
// dev_audio.c 内部
static const hal_i2s_ops_t *s_i2s;
static const hal_gpio_ops_t *s_gpio;

static esp_err_t s_dev_audio_init(const dev_audio_config_t *cfg, dev_audio_handle_t *out) {
    s_i2s = hal_i2s_get_ops();
    s_gpio = hal_gpio_get_ops();
    // ...
    s_i2s->init(&i2s_cfg);
    // ...
}
```

- HAL ops 指针在 Device 内部，上层不可见
- Application 层不可直接调用 HAL

### 6. dev_init_all 聚合

配置参数硬编码在 `dev_init.c` 内，`main.c` 不可见：

```c
// main.c
void app_main(void) {
    ESP_ERROR_CHECK(nvs_flash_init());

    dev_handles_t handles;
    ESP_ERROR_CHECK(dev_init_all(&handles));  // 配置在 dev_init.c 内
    ESP_ERROR_CHECK(app_init_all(&handles));  // handles 传递给 App 层

    ESP_LOGI(TAG, "Music Box started.");
}
```

初始化顺序即依赖顺序：I2C 总线 → Audio → LED Strip → Light Sensor → Display → Button

### 7. 模块 API 速查

| 模块 | 头文件 | Config 关键字段 | Ops 核心方法 |
|------|--------|----------------|-------------|
| `dev_audio` | `dev_audio.h` | port, bclk/ws/dout/mclk/sd pins, sample_rate | init, deinit, play, pause, resume, stop, set_volume, get_state |
| `dev_led_strip` | `dev_led_strip.h` | gpio_num, resolution_hz, num_pixels, bit timing | init, deinit, set_pixel, refresh, clear, set_brightness |
| `dev_light_sensor` | `dev_light_sensor.h` | adc_unit, adc_channel, atten, bitwidth, filter_window | init, deinit, read_raw, read_lux |
| `dev_display` | `dev_display.h` | i2c_port, device_address, scl_speed_hz, width, height | init, deinit, clear, show_text, set_contrast, display_on, display_off |
| `dev_button` | `dev_button.h` | pins[].pin, long_press_ms, debounce_ms | init, deinit, register_callback |
| `dev_init` | `dev_init.h` | handles 聚合结构体 | dev_init_all(out) |

### 8. 事件驱动接口

对于按键等异步事件，Device 层使用回调模式：

```c
// dev_button.h
typedef enum {
    DEV_BTN_EVT_SHORT_PRESS = 0,
    DEV_BTN_EVT_LONG_PRESS,
} dev_button_event_t;

typedef void (*dev_button_cb_t)(gpio_num_t pin, dev_button_event_t event, void *arg);

// 注册回调，事件触发时回调被调用
ops->register_callback(h, my_callback, my_arg);
```

- 回调在 ISR 上下文外执行（通过 FreeRTOS 队列或定时器回调转发）
- 每个模块最多一个回调（单 handle 模式下的简化）
- 回调 arg 由调用方提供，Device 不负责解释
