# music_box 编码规范

> 继承自 GERB_GSI 编码规范，按项目规模裁减适配。

---

## 1. 命名规则

```
函数         层前缀_模块_动作        hal_i2c_write_reg()
                                     dev_audio_play()
                                     app_player_next_track()

类型         模块_描述_t             dev_audio_config_t
                                     app_player_state_t

枚举值       模块_描述_值            DEV_BTN_EVT_SHORT_UP

宏/常量      全大写+下划线           LED_STRIP_NUM_PIXELS

静态变量     s_ 前缀                 static bool s_is_playing;

日志TAG      每个.c顶部定义          static const char *TAG = "DEV_AUDIO";
```

### 示例

```c
// dev_audio.h
typedef struct dev_audio_s *dev_audio_handle_t;

typedef struct {
    i2s_port_t     i2s_port;
    gpio_num_t     bclk_pin;
    gpio_num_t     lrc_pin;
    gpio_num_t     din_pin;
    gpio_num_t     sd_pin;
    uint32_t       sample_rate;
} dev_audio_config_t;

typedef struct {
    esp_err_t (*init)(const dev_audio_config_t *cfg, dev_audio_handle_t *out);
    esp_err_t (*deinit)(dev_audio_handle_t h);
    esp_err_t (*play)(dev_audio_handle_t h, const char *filepath);
    esp_err_t (*stop)(dev_audio_handle_t h);
    esp_err_t (*set_volume)(dev_audio_handle_t h, uint8_t vol); // 0~100
} dev_audio_ops_t;

const dev_audio_ops_t *dev_audio_get_ops(void);
```

## 2. ops + handle 模式 (核心设计模式)

每个 Device 模块必须遵循此模式：

- **handle** = 不透明指针 (`typedef struct xxx_s *xxx_handle_t`)，隐藏内部状态
- **ops** = 函数表结构体，通过 `_get_ops()` 单例获取
- 所有操作通过 `ops->method(handle, ...)` 调用
- `.c` 内所有函数 `static`，仅 ops 表暴露

```c
// 调用方
const dev_audio_ops_t *audio = dev_audio_get_ops();
dev_audio_handle_t h;
audio->init(&cfg, &h);
audio->play(h, "/music/track1.mp3");
audio->deinit(h);
```

## 3. 错误处理

```
- 所有函数返回 esp_err_t (void 函数例外: on/off/stop 等简单动作)
- 失败时 ESP_LOGE(TAG, "...") 后返回错误码
- NULL 指针检查: 返回 ESP_ERR_INVALID_ARG
- 关键初始化失败用 ESP_ERROR_CHECK() — 初始化失败重启即可
```

```c
static esp_err_t dev_audio_init(const dev_audio_config_t *cfg,
                                dev_audio_handle_t *out) {
    if (cfg == NULL || out == NULL) {
        return ESP_ERR_INVALID_ARG;
    }
    dev_audio_handle_t h = calloc(1, sizeof(*h));
    if (h == NULL) {
        ESP_LOGE(TAG, "OOM");
        return ESP_ERR_NO_MEM;
    }
    // ...
    *out = h;
    return ESP_OK;
}
```

## 4. 类型无关接口

.h 文件不出现具体芯片型号，所有参数通过 config 结构体传入：

```c
// ✅ 正确: dev_display.h
esp_err_t dev_display_init(const dev_display_config_t *cfg);
esp_err_t dev_display_show_text(const char *line1, const char *line2);

// ❌ 错误: 绑定型号
esp_err_t ssd1306_show_text(const char *text);

// ✅ 正确: HAL层参数化
hal_i2c_config_t cfg = {
    .port      = I2C_NUM_0,
    .sda_pin   = GPIO_NUM_41,
    .scl_pin   = GPIO_NUM_42,
    .clk_speed = 400000,
};
hal_i2c_init(I2C_NUM_0, &cfg);

// ❌ 错误: 硬编码引脚
void hal_i2c_init_for_oled(void) {
    i2c_config.sda_io_num = 41; // 硬编码!
}
```

## 5. .h 引用规则

```
.c 只包含自身对应的 .h
.h 负责包含自身依赖的其他 .h
```

```c
// dev_audio.h — 自己包含依赖
#include "hal_i2s.h"
#include "hal_gpio.h"

// dev_audio.c — 只包含自己
#include "dev_audio.h"
```

---

## 6. 层内聚原则

```
- 配置参数集中在该层的 _init_all() 内，main.c 不可见
- HAL 层由 Device 层按需拉起，不独立聚合
- 共享总线 (如 I2C_NUM_0) 在 dev_init_all() 开头显式初始化一次
- 初始化顺序即依赖顺序
```

## 7. 日志规范

```c
// 每个 .c 顶部
static const char *TAG = "MODULE_NAME";

// 日志级别
ESP_LOGE(TAG, "error message: %s", esp_err_to_name(ret));  // 错误
ESP_LOGW(TAG, "warning: volume clipped to %d", vol);        // 警告
ESP_LOGI(TAG, "audio initialized, sample_rate=%lu", rate);  // 信息
ESP_LOGD(TAG, "buffer filled: %d bytes", len);              // 调试
```

---

## 8. 内存管理

```
- 优先使用 PSRAM (heap_caps_malloc(..., MALLOC_CAP_SPIRAM))
- 音频缓冲区 → PSRAM (8MB PSRAM 充裕)
- 小对象 (handle 结构体) → calloc (内部 SRAM)
- 分配失败必须检查 + 记录日志 + 返回 ESP_ERR_NO_MEM
- delete/deinit 中 free 后不置 NULL (handle 由调用方管理)
```

---

## 9. 禁止事项

```
❌ 在 .h 中出现芯片型号 (SSD1306, MAX98357A, WS2812B)
❌ 在 .c 中硬编码 GPIO 编号 (必须从 config 传入)
❌ Application 直接调用 HAL 层 (定时器管理除外)
❌ 同一引脚号在多处重复定义
❌ 为不可能发生的场景写错误处理
❌ 为未要求的 feature 预留接口
```
